# Test Notes

There is no regular unit test suite here. Build the changed menu, then test that path on hardware.

## Build Checks

```sh
make vshmenu
make arkmenu
make xmenu
```

For shared code or build-rule changes:

```sh
make
```

Expected outputs:

```text
dist/VSHMENU.PRX
dist/VBOOT.PBP
dist/XBOOT.PBP
dist/THEME.ARK
```

## Hardware Folder

```text
PSP/SAVEDATA/ARK_01234/
```

Back up files before replacing them.

## VSH Menu

Install:

```text
dist/VSHMENU.PRX -> PSP/SAVEDATA/ARK_01234/VSHMENU.PRX
```

Check:

- Boot ARK and return to XMB.
- Press `SELECT`.
- Open and close the menu.
- Change one harmless setting.
- Reopen the menu and confirm it persisted.

For language or font changes, install matching `VSHMENU.PRX` and `LANG.ARK`.

Check startup, entry scanning, menu navigation, launching a known working entry, and exit behavior.

## Language Regressions

For `vshMenu/lang.c`, check byte format before changing parsing code.

- Non-CJK satellite files are plain byte strings.
- CJK satellite files generated by ARK use low byte values as `.pf` glyph indexes.
- CR and LF are safe line endings. Other values below `0x20` may still be glyphs.

Square-looking CJK glyphs usually belong in ARK language resources:

```text
ARK-5/Resources/Language/quan.bdf
ARK-5/Resources/Language/Translations/resources/satelite_*_utf8.txt
```
